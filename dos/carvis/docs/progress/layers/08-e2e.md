# 08-e2e — 端到端集成验收

## 概述

E2E 层是 Carvis 的最终集成验证层。不引入新的业务逻辑，仅编排已有模块完成完整链路测试。

## 文件结构

```
src/e2e/
├── smoke.ts    # 端到端集成 smoke
└── index.ts    # barrel export
```

## smoke.ts — 端到端集成测试

8 项断言，逐一验证集成链路：

| # | 断言 | 验证点 |
|---|---|---|
| 1 | submitTask 返回有效 runId，phase=created | 入口 |
| 2 | advance 推进至 shutdown，≤20 次 | 全阶段推进 |
| 3 | 核心阶段可观测：parallel_roles_working / engineer_building / output_ready / retaining_agents / shutdown | 阶段流 |
| 4 | 5 角色 agent 全部创建（manager/writer/artist/researcher/engineer） | Agent 池 |
| 5 | output.ready 事件携带正确 outputPath | 事件广播 |
| 6 | manifest.json + report.md 存在且内容正确 | 文件产物 |
| 7 | shutdown 后 queue=0, active=0, idle=0 | 清理 |
| 8 | heartbeat 正常发布 | 心跳 |

## 设计决策

- **manager_planning 暂态不可观测**：scheduler 在单次 `advance()` 内完成 `phase=manager_planning → executeSequential("manager") → phase=parallel_roles_working`，外部调用者看不到 `manager_planning`。E2E 不将其列入必须观测阶段。
- **使用临时目录**：所有 workplace/output 写入 `os.tmpdir()`，finally 块清理，不污染项目目录。
- **独立脚本模式**：与已有 smoke 一致，顶层代码直接执行，不导出函数。

## npm 脚本

- `npm run e2e:smoke` — 单独运行 E2E smoke
- `npm test` — 运行全部 8 个 smoke（含 e2e）
